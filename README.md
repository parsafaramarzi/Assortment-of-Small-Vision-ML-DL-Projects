## 4. Optical Character Recognition (OCR) Comparison
**Files:** 1. `EasyOCR-TextRecognition.py`
2. `PaddleOCR-TextRecognition.py`
3. `Hybrid-TextRecognition.py`

This section compares three different approaches to extracting text from images. By implementing a **Hybrid** pipeline, we combine the strong text *detection* of EasyOCR with the superior text *recognition* of PaddleOCR.

| Strategy | File | Description |
| :--- | :--- | :--- |
| **EasyOCR** | `EasyOCR...py` | Uses a GPU-accelerated lightweight model. Good at detecting where text is, but sometimes struggles with complex fonts. |
| **PaddleOCR** | `PaddleOCR...py` | Industrial-grade OCR. Excellent at handling text orientation and diverse languages. |
| **Hybrid** | `Hybrid...py` | **Best of Both Worlds:** Uses EasyOCR to find the bounding boxes, then crops those areas and feeds them to PaddleOCR for final text reading. |

**Comparative Results:**

| EasyOCR Only | PaddleOCR Only | Hybrid (EasyOCR + Paddle) |
| :---: | :---: | :---: |
| <img src="output/text_rec_easyocr01.jpg" width="100%" alt="EasyOCR Result"/> | <img src="output/text_rec_paddleocr_only.jpg" width="100%" alt="PaddleOCR Result"/> | <img src="output/text_rec_hybrid_crop01.jpg" width="100%" alt="Hybrid Result"/> |
| *Fast detection, decent recognition.* | *High accuracy, different bounding logic.* | *Accurate boxes from EasyOCR, cleaned text from Paddle.* |

> **Tech Stack:**
> * **EasyOCR:** PyTorch backend, supports 80+ languages.
> * **PaddleOCR:** Ultra-lightweight PP-OCRv3/v4 models.
> * **Hybrid Logic:** `Crop -> Predict -> Fallback` mechanism implemented manually.
